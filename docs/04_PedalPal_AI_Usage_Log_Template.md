# AI Usage Log

## 1. Overview
- Purpose: Document how AI tools were used during the PedalPal modernization.
- Tools: ChatGPT, Copilot, repository-specific AI skills.

## 2. AI role in the project
- Code refactoring and architecture planning.
- UI/UX redesign and wireframe generation.
- Documentation and delivery artifact creation.

## 3. Prompts used
- Prompt 1: "Audit this legacy bike rental PHP module and recommend a Laravel-style modernization plan."
- Prompt 2: "Generate a component-driven Vue 3 architecture for the PedalPal shop pages."
- Prompt 3: "Write a README template that includes AI usage details, assumptions, and integration notes."

## 4. AI-driven outputs
- Legacy code analysis and modernization recommendations.
- PRD and architecture doc structure.
- Wireframe documentation and UX spec outlines.
- Documentation templates and final deliverable structure.

## 5. How AI was guided
- Provided the legacy repository structure and assignment goals.
- Asked for evaluation-focused outcomes, maintainability improvements, and UX modernization.
- Used follow-up prompts to refine architecture, docs, and component designs.

## 6. Lessons learned
- AI is effective for generating structured documentation and design artifacts.
- Human validation is required to align architecture decisions with real application constraints.
- Clear prompts and repository-specific guidance improved output relevance.

## 7. Appendix: Additional prompts
- "Suggest an API contract for managing bike inventory, rentals, and accessories."
- "Create a wireframe annotation guide for the rental confirmation flow."
- "Write a project assumptions and trade-offs section for the final README."
